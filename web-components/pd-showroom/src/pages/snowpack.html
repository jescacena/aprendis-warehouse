Snowpack

Dentro del package.json hay un script de postinstall que llama a snowpack y se ejecuta cada vez que hacemos npm install, ya sea porque es la primera instalación o porque hemos traido código nuevo del repositorio.

Cuando añadimos un módulo nuevo mediante npm install nombre_componente también es conveniente hacer un npm install

Las tareas que realiza snowpack basicamente son exponer dentro del servidor de de desarrollo los módulos ESM que queramos usar en nuestra aplicación. Para ello copia los módulos requeridos de node_modules a la carpera src/web_modules. En caso de que hiciera falta también realiza las tareas de transpilación.

Si usas webpack u otra herramienta de bundling normalmente puedes importar módulos de node_modules sin indicar la ruta relativa al fichero y sin la extensión del fichero: import {LitElement, html} from 'lit-element';. Pero esta manera de realizar la importación no funciona en javascript vanilla sin el uso de alguna de las herramientas mencionadas anteriormente, ya que la ruta al fichero que quieres importar tiene que ser exacta y normalmente no tenemos acceso a node_modules porque esta fuera de la carpeta src.

Snowpack nos permite acceder mediante import al recurso que queramos usando la forma normal de javascript ya que expone los recursos que necesitamos de node_modules en la carpeta src/web_modules: import {LitElement, html} from './web_modules/lit-element.js';
